# iOS App Store Optimization

Good App Store optimization starts with a real product, not keyword stuffing.

## Optimize In This Order

1. product clarity
2. icon and screenshot conversion quality
3. title, subtitle, and keyword strategy
4. description and promotional text
5. review and rating trust signals

## Metadata Rules

- write for the right user before writing for search
- make the title and subtitle concrete, not clever
- use keywords that describe the real job of the app
- do not claim unfinished flows, hidden features, or vague AI magic
- remove repetition that wastes limited metadata space

## Conversion Rules

- the icon should help someone notice the app
- the first screenshot should explain the core value immediately
- screenshot copy should be faster to understand than the UI itself
- the listing should match the first-run experience, not oversell it

## Positioning Rules

- lead with the primary use case, not a long feature list
- speak in user language, not builder language
- choose one promise the listing can own cleanly

## Avoid

- generic "all-in-one" positioning
- stuffing the same keyword everywhere
- descriptions that read like release notes
- screenshot headlines that are too clever to scan
- ASO changes that make the product feel less trustworthy
