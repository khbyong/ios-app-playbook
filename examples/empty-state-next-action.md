# Empty State Next Action

## Problem

An empty dashboard explained what the feature was, but did not clearly tell the user what to do next.

## Weak Version

- icon
- title: `No expenses yet`
- subtitle: `Track your daily spending here.`

## Stronger Version

- icon
- title: `No expenses logged today`
- subtitle: `Start with your first expense so today's total, budget, and trends can appear here.`
- primary action: `Log Expense`

## Why It Is Better

- it anchors the empty state to the user's current moment
- it names the next action explicitly
- it connects the action to the value unlocked by taking it

## Playbook Skills Behind It

- `ios-empty-states-and-first-run`
- `swiftui-product-ui`
