# How To Prep An Indie iOS App For App Store Review

Review prep is mostly about removing ambiguity.

## Validate The Build First

- the app launches reliably
- the core loop works on device
- no unfinished feature is advertised as live
- warnings tied to runtime behavior are fixed

## Align Metadata

- title and subtitle match the product
- screenshots show real behavior
- description does not overclaim
- promo text reflects what shipped

## Remove Review Traps

- unused capabilities
- incomplete privacy manifest coverage
- sync wording that implies restore when none exists
- disabled features that still look active

## Final Question

If a reviewer misunderstands what the app does, whose fault is that?

Usually the product, copy, or settings UX made the wrong thing look true.
